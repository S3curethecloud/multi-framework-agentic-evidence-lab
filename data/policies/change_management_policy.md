# Change Management Policy

**Document ID:** POL-CHANGE-001
**Control area:** Change Management
**Mock owner:** Engineering Governance
**Mock effective date:** 2026-01-20
**Dataset status:** Mock evidence only

## Purpose

This mock policy defines change-approval expectations for the evidence-review lab.

## Requirements

1. Production changes must reference a ticket.
2. Production changes must include an implementation owner.
3. Production changes must include at least one approval before deployment.
4. Higher-risk changes require secondary approval.
5. Deployment records must include timestamp, target environment, rollback notes, and change-ticket linkage.
6. Emergency changes must include post-change review evidence.

## Evidence sufficiency rule

Change-management evidence is sufficient only when approval, deployment, rollback, and ticket records can be linked.

## Boundary

This policy is mock readiness evidence for a lab. It does not prove production operating effectiveness or SOC 2 certification.
