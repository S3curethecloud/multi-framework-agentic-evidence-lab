# Incident Response Policy

**Document ID:** POL-IR-001  
**Control area:** Incident Response  
**Mock owner:** Security Operations  
**Mock effective date:** 2026-02-01  
**Dataset status:** Mock evidence only

## Purpose

This mock policy defines incident-response expectations for the evidence-review lab.

## Requirements

1. Incidents must be triaged by severity.
2. Incident records must identify owner, timeline, containment, eradication, recovery, and lessons learned.
3. Incident-response tabletop exercises should be performed annually.
4. Incident evidence must preserve timestamps, participants, actions taken, and closure notes.

## Known Phase 2 gap

The policy exists, but Phase 2 does not include tabletop exercise evidence. An agent should identify this as a gap when asked about incident-response evidence sufficiency.

## Boundary

This policy is mock readiness evidence for a lab. It does not prove production operating effectiveness or SOC 2 certification.
